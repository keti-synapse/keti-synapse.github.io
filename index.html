Abstract

하나의 메시지 표준, 두 가지 운용 모드

status·command를 통일하면, 어떤 관제 화면이든 모든 로봇을 같은 인터페이스로 다룰 수 있다.

SYNAPSE는 모바일·매니퓰레이터·휴머노이드 등 이기종 멀티로봇을 단일 메시지 표준(synapse)으로 추상화하는 플랫폼이다. 로봇에 tom_and_gerri를 설치하는 것만으로 표준 status/command가 자동으로 노출되며, 별도 서버 연동 작업 없이 1:1 원격조작(COCKPIT)과 N:N 플릿 관제(NEXUS)가 하나의 코드베이스 위에서 동시에 활성화된다.

설계의 중심에는 클린 아키텍처가 있다. 모든 컴포넌트는 가장 안쪽의 synapse 표준을 import하고, 의존성은 언제나 안쪽을 향한다. 전송 계층(WebRTC), GUI, 데이터베이스, 로봇 SDK는 모두 바깥의 교체 가능한 세부사항이다. 상태의 원본은 로봇(엣지)이 소유하고, 서버는 그 사본(뷰)만 캐시한다 — Edge-Authoritative 원칙이다.

Motivation

왜 이 프로젝트인가 — 마주한 문제

이 스프린트는 비용이 크지만, 얻는 것이 더 크다.SYNAPSE는 기술 표준과 팀 문화 둘 다를 겨냥한다.

PROBLEM 01
🏝️
정보 섬 (Information Islands)
rubberneck, tom_and_gerri, solar … 각 프로젝트가 작은 섬을 만든다. 섬마다 정보가 따로 흐르고 서로 연결되지 않는다.

파편화 — 같은 일이 흩어짐
중복 — 같은 정보를 따로 보관
왜곡 — 섬마다 조금씩 달라짐
PROBLEM 02
🔀
코드 패턴 불일치
호환성보다 "일단 되게"에 급급한 결과 패턴이 제각각이다.

예 — MCAP 빌더: dict로 status를 관리하다 MCAP 저장을 위해 Pydantic으로 바꿨지만 내부는 여전히 dict. 결국 로봇마다 Pydantic·dict 혼재.

→ 이해·확장 비용 폭증. 안 쓰는 코드는 또 하나의 섬.
SOLUTION
🤝
표준 + 문화, 둘 다
Technical: synapse가 메시지·status·자료구조를 통일해 섬을 잇는다. standardize()가 dict·Pydantic 모두 수용 → 전면 재작성 없이 점진 마이그레이션.

Cultural: 주(2주) 단위 랩 세미나로 최신 기술·큰 변경점을 공유하고 함께 브레인스토밍한다.
핵심: 표준이 섬을 잇고, 소통이 표준을 살아있게 한다.최소 한 명 이상은 이 SW 아키텍처를 전부 이해하고, 수요에 맞게 필요/불필요를 조정할 수 있어야 한다.
Key Concept

올인원 — 설치 즉시 플릿의 일부

로봇에 tom_and_gerri를 설치하면 synapse 표준 status/command가 자동 노출된다.별도 작업 없이 NEXUS 플릿에서 즉시 관제 가능 — 한 번의 설치로 1:1 + N:N 동시 ON.

🗄️
플릿 자동 관제
NEXUS가 즉시 인식. 1:1 COCKPIT 조작과 N:N 플릿 관제가 동시에 활성화된다.
Architecture

클린 아키텍처 — 의존성은 안쪽으로

바깥은 교체 가능한 세부사항, 안쪽은 안정적인 표준.의존 화살표는 항상 바깥에서 안쪽을 향하며, 안쪽은 바깥을 모른다.

🗄️ DB in-memory → SQLite
의존 방향 (바깥 → 안쪽). 안쪽은 바깥을 모른다.

3-Way Hybrid 통신

NEXUS는 시그널링 중계와 대시보드 캐시만 담당한다.실제 제어·미디어 데이터는 WebRTC P2P로 직결되어 NEXUS를 거치지 않는다.

Mission System

3계층 미션 구조

"실패는 정상, 인간 개입 우선."FleetMission → RobotMission → Task 순으로 위임한다.Python 함수·JSON·REST 모두 지원.

1
FleetMission
다중 로봇 조율. FleetTask 그래프(DAG)를 depends_on으로 정의. 설계상 NEXUS가 조율 담당(P3 예정), 로봇 점유는 TOM 책임.
NEXUS
▼ RobotMission 위임
2
RobotMission
단일 로봇의 Task 시퀀스 실행. MissionRunner가 관리. 상태는 로봇 측 보유 → 재접속 시 자동 복구. 개입: NEED_HELP → TELEOP → resume/abort.
GERRI
▼ Task 실행

FleetMission — 커피 배달 예시 (DAG)

synapse.missions 헬퍼
def coffee_delivery() -> FleetMission:
    return FleetMission(
        name="coffee_delivery", site="itp",
        tasks={
            "go":       call("mobile_01", "go_to_coffee"),
            "make":     call("coffee_01", "make_americano",
                          depends_on=["go"]),
            "transfer": call("arm_01", "transfer_cup",
                          depends_on=["make"]),
            "deliver":  call("mobile_01", "go_to_dest",
                          depends_on=["transfer"]),
        })

RobotMission — 5층→1층 배달

synapse.missions 헬퍼
def delivery_5f_to_1f() -> RobotMission:
    return RobotMission(name="delivery", tasks=[
        cap("undock", "undock"),
        cap("go", "navigate_to_pose", x=1.0, y=2.0),
        wait_input("confirm", prompt="도착했나요?",
                   buttons={"yes": "예"}),
        cap("dock", "dock"),
        notify("done", "배달 완료"),
    ])
